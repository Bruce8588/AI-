### 详细技术记录

#### 1. Nginx 删除
```bash
yum remove -y nginx nginx-common nginx-all-modules nginx-filesystem
```

#### 2. 端口修改

交易后端 `app.py` 最后一行的 `app.run()`:
```python
app.run(debug=True, host='0.0.0.0', port=5649)
```

星夜前端代理 `/root/life-manager/frontend/vite.config.js`:
```javascript
proxy: {
  '/api': {
    target: 'http://localhost:5649',  // 原为 5001
    changeOrigin: true
  }
}
```

#### 3. 数据丢失问题修复

**症状：** 从星夜跳转交易系统无数据
**原因：** 星夜代理指向旧端口 5001，但后端已在 5649
**修复：** 更新 vite.config.js 的 proxy target

#### 4. systemd 服务配置

`/etc/systemd/system/life-manager.service`:
```ini
[Unit]
Description=Life Manager Frontend (Starry)
After=network.target

[Service]
Type=simple
WorkingDirectory=/root/life-manager/frontend
ExecStart=/root/life-manager/frontend/node_modules/.bin/vite --host --port 8975
Restart=always
RestartSec=10
User=root

[Install]
WantedBy=multi-user.target
```

`/etc/systemd/system/trading-frontend.service`:
```ini
[Unit]
Description=Trading System Frontend
After=network.target

[Service]
Type=simple
WorkingDirectory=/root/life-manager/trading-system/frontend
ExecStart=/root/life-manager/trading-system/frontend/node_modules/.bin/vite --host --port 4816
Restart=always
RestartSec=10
User=root

[Install]
WantedBy=multi-user.target
```

应用配置：`systemctl daemon-reload && systemctl restart life-manager trading-frontend`

#### 5. stunnel HTTPS 配置

安装：`yum install -y stunnel`

生成证书：
```bash
openssl req -x509 -newkey rsa:2048 \
  -keyout /tmp/trading-key.pem \
  -out /tmp/trading-cert.pem \
  -days 365 -nodes \
  -subj "/CN=8.129.109.139" \
  -addext "subjectAltName=IP:8.129.109.139"
```

配置文件 `/tmp/stunnel-trading.conf`:
```ini
foreground = no
pid = /tmp/stunnel.pid
[https]
accept = 443
connect = 127.0.0.1:4816
cert = /tmp/trading-cert.pem
key = /tmp/trading-key.pem
```

启动：`stunnel /tmp/stunnel-trading.conf`
