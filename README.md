# AI对话 - 双AI协作交接规范

## 文档目的

本文档是 **本地AI（西琳）** 与 **服务器AI（艾莉丝/Iris）** 之间的信息互通规范。

通过共享此文档，确保两个AI能够：
1. 理解彼此的工作范围和职责
2. 通过 GitHub 传递重要信息
3. 记录工作日志，确保信息不遗漏

---

## 核心规则

### 规则一：文档即契约
所有在此仓库中的文档，是两个AI共同的「契约」。双方必须遵守文档中的所有规则，不得擅自更改规则（除非双方协商一致）。

### 规则二：工作后必记录
每次完成重要工作后，必须在工作日志中记录。时间戳以 GMT+8 为准。

### 规则三：信息同步优先级
- GitHub 是唯一真相源
- 双方每次启动时，先拉取最新版本
- 重要信息必须 commit + push，不得只存在本地

## 信息互通流程

### 场景一：A完成工作 → 通知B

1. A 完成工作，commit + push 到本仓库
2. A 在 `工作日志.md` 中记录完成的工作
3. A 在 `待办交接.md` 中写下需要B关注的事项（如果有）
4. B 下次启动时，pull 最新版本，读取 `工作日志.md` 和 `待办交接.md`

### 场景二：A需要B协助

1. A 在 `待办交接.md` 中写下需求
2. 明确标注：「需要[AI名称]处理」
3. B 读取后处理，完成后在工作日志中记录

### 场景三：B完成工作 → 通知A

1. B 完成工作，push 到本仓库
2. B 在 `工作日志.md` 中记录完成的工作
3. A 下次启动时，pull 获取
